# OpenSpec Decision Clarification

Applies when creating or updating OpenSpec plans, proposals, designs, tasks, or other change artifacts.

Rule:
- If there is a discrepancy, ambiguity, or competing option about technology, architecture, scope, or any other material decision, do not assume a choice in the OpenSpec artifact and ask the user immediately.
- If the requested decision is clearly weaker than a better available option, say so directly, explain the better option briefly, and wait for the user's response before continuing.
- If the user confirms they still want the weaker option, respect that decision and proceed without repeating the objection.

Notes:
- This rule is about decision points inside OpenSpec work, not routine implementation details that are already clearly specified.
- Prefer one short clarification question over embedding an unconfirmed assumption in the plan or change.
